## 2. 시나리오 규약

- ID는 S1~S9. 각 시나리오는 4절 구조: **유저 스토리**(누가·언제·왜) / **화면 흐름** / **시스템 동작**(코드 심볼 매핑) / **수용 기준**(검증 가능한 문장).
- 시스템 동작의 심볼: `QueryExtractor`(`src/eddr/query/extract.py` 신규 — gemma4:e2b 구조화 추출) · `QueryService`(`src/eddr/query/tools.py` — 내부 검색 서비스, ADR-0009로 LLM tool 아님) · `NominatimClient`(`src/eddr/geocode/nominatim.py`) · `EddrDatabase`(`src/eddr/db/repository.py`) · 서버 라우트(`src/eddr/server/routes/`).
- **채팅은 없다**(ADR-0009). 질의의 출력은 항상 사진이며, 텍스트 응답·후속 질문 맥락은 제품 범위 밖.
- 마일스톤(M0~M6)은 [prd.md §7](prd.md).

## 3. 시나리오

### S1. 지도 홈 — 현위치와 내 사진 (핵심 진입점)

- **유저 스토리** — 외출 중 카페에서 앱을 연다. 지도가 현위치를 중심으로 열리고, 주변에 예전에 찍었던 사진들이 마커로 떠 있다. "여기 작년에도 왔었네" — 마커를 탭해 그날 사진을 본다.
- **화면 흐름** — 전체 화면 지도 + 현위치 점 → 사진 위치 마커(저줌 = 개수 클러스터 원, 고줌 = 썸네일 마커) → 마커/클러스터 탭 → 해당 날짜 사진(S3의 날짜 상세) → 라이트박스. 하단에 검색창(S2) 상시 노출, 상단 우측에 위치 미상 배지(S4).
- **시스템 동작** — `GET /api/map/photos`(노출 GPS 5,587점 GeoJSON 1회 전량) → MapLibre cluster source. 고줌(≥14)에서만 화면 내 비클러스터 점에 HTML 썸네일 마커(상한 ~60, `thumb?size=320` 재사용). 현위치는 `navigator.geolocation.watchPosition` — **HTTPS 필수**(S7), 거부·실패 시 최근 GPS 사진 위치로 폴백.
- **수용 기준** — 폰(tailnet HTTPS)에서 현위치 중심 지도 기동. 지도 데이터 첫 페이로드(타일 제외) < 500KB. 클러스터 탭 → 날짜 상세 → 라이트박스 → 원본 저장까지 무중단. 좌표는 내 서버→내 브라우저만 흐름(ADR-0009 §3).

### S2. 자연어 검색 — 사진만 추천 (날짜 lane)

- **유저 스토리** — 가족과 저녁을 먹다 "몽골에서 은하수 본 게 언제였지?"가 나왔다. 하단 검색창에 "몽골 은하수"를 입력한다. 답변 텍스트 없이, 그날의 사진들이 날짜별 한 줄(lane)로 올라온다 — 가장 그럴듯한 날짜가 맨 위.
- **화면 흐름** — 하단 검색창 입력 → 결과 시트(bottom sheet)가 올라옴 → **날짜별 1 lane**: 헤더(날짜·장소·장수) + 가로 스크롤 스트립 — 접힘 상태 최대 5장, 뷰포트에는 약 3장 보임(marquee), 5번째 뒤 **'더보기'** 셀 → S3. lane 정렬은 관련도순. 시트 상단에 **해석 칩**("몽골 · 은하수" 등 추출 결과 — 오해석을 사용자가 즉시 인지). 지도에는 결과 사진 위치가 하이라이트되고 최상위 lane으로 flyTo.
- **시스템 동작** — `POST /api/search {query}` → `QueryExtractor.extract()`(gemma4:e2b structured output: `keywords_en`·`date_from/to`·`countries`·`cities`, 오늘 날짜 KST 주입, 실패 시 임베딩-only 폴백) → `QueryService.semantic_search_photos(query=원문, keywords=keywords_en, trip_ids=지역명→trips 매칭, …)`(Chroma+BM25 RRF — 검증 자산 그대로) → 서버에서 KST 날짜 그룹핑·관련도순 정렬 → `{interpretation, groups[]}`(사진 좌표 동봉). 외부 LLM 호출 0회(ADR-0009 §2). ollama 미기동 → 503 + 안내 토스트.
- **수용 기준** — 골든셋 v2 **자동 채점**(사진형 = 정답 사진이 결과 포함, fact형 G04·G08 = 정답 날짜 lane 상위 노출): **G06 제외 9문항 중 8↑**(M3 게이트). 해석 칩이 추출 결과·폴백 여부를 항상 표시. ollama 다운 시 명확한 한국어 안내.

### S3. 더보기 → 날짜 상세 — 시트 축소·지도 확대

- **유저 스토리** — 은하수 lane의 '더보기'를 누른다. 그 날짜의 모든 사진이 펼쳐지고, 위쪽 지도는 그날 사진들의 위치로 확대된다 — "이 길을 따라 이동하며 찍었구나"가 한눈에 보인다.
- **화면 흐름** — lane '더보기'(또는 S1 마커 탭) → **날짜 상세**: 결과 시트는 하단으로 축소, 지도가 그 날짜 사진들 범위로 확대(fitBounds), 시트에 해당 날짜 전체 그리드 → 사진 탭 → 라이트박스(스와이프·핀치줌·상세 패널: 촬영일·장소·캡션·메모 S5) → "원본 보기/저장". 그날 GPS 사진이 0장이면 "위치 지정하기" 링크 → S4 진입.
- **시스템 동작** — `GET /api/photos/by-date?date=`(해당 KST 날짜의 노출 사진 전부, 좌표 동봉 — 실측 일 최대 155장) · 라이트박스는 `GET /api/photos/{id}`(좌표·메모 포함으로 확장) + `thumb?size=1280` + `original`(기존 그대로 — photo_id 간접 서빙, ADR-0008).
- **수용 기준** — HEIC 원본 사진도 lane → 라이트박스 → 원본 저장 3단계 동작(v1 S2 계승). 응답 어디에도 파일시스템 절대경로 미노출. 날짜 상세 진입 시 지도 fitBounds가 해당 날짜 GPS 사진을 모두 포함.

### S4. 위치 미상 해소 — 수동 지오코딩 (빨간 느낌표)

- **유저 스토리** — 화면 상단 우측의 빨간 느낌표가 거슬린다. 눌러 보니 위치 없는 사진들이 날짜별로 묶여 있다. "2018-04-22 · 16장" 그룹 — 개심사 벚꽃 사진들이다. 그룹을 열고 "서산 개심사"를 검색하자 지도가 후보지로 날아간다(flyTo). 후보 핀을 탭하고(또는 지도를 움직여 정확한 자리를 꾸욱 눌러) "이 날짜 16장 전체에 적용" 확인 모달을 승인한다. 지도에 새 마커들이 돋아난다.
- **화면 흐름** — 빨간 느낌표 배지(잔여 그룹 수) → 일별 그룹 드로어(날짜 DESC, 카드 = 날짜·N장·대표 썸네일 4·trip 힌트, 진행 표시) → 그룹 선택 → **장소 검색 입력**(자연어·다국어) → 후보 ≤5를 지도 핀 + 리스트로 → 후보 탭 = flyTo(zoom 14) → "여기로 지정" 버튼 **또는** 지도 아무 곳 **long-press**(650ms) → **확인 모달**("이 날짜 N장 전체 적용" + 대표 썸네일 + 주소 미리보기/"저장 후 주소 자동 조회") → 저장 → 드로어에서 그룹 제거 + 지도 마커 갱신.
- **시스템 동작** — `GET /api/photos/no-location`(노출 조건 + `latitude IS NULL AND taken_at IS NOT NULL`, KST 일별 그룹 — 실측 **2,867장/525그룹** — D26 확정 시 521, M1 KST 정규화로 날짜 경계 이동, trip 힌트 757장) · `GET /api/geocode/search?q=`(`NominatimClient.search()` 신규 — **서버 프록시 필수**, accept-language=ko, 1 req/s) · `PUT /api/photos/location {photo_ids[], latitude, longitude}` → `update_photo_location(…, source='manual')` + 기존 reverse 경로(양자화 캐시→`reverse`→`update_photo_geo`)로 country/city/district 채움(ADR-0009 §4). trips 자동 재계산 없음 — 완료 토스트에 `eddr trips recompute` 안내.
- **수용 기준** — 개심사 16장 그룹에 위치 지정 → 골든셋 **G06 통과 = 10/10**(M4 게이트, 구 D24 enrichment 게이트의 대체). 저장 후 해당 좌표에 마커 노출·배지 카운트 감소·DB `location_source='manual'` 기록. Nominatim 후보 0건이어도 long-press 직접 지정은 항상 가능. 폰에서 스크롤 중 long-press 오발화 없음. 날짜조차 없는 764장은 v1 범위 밖(백로그).

### S5. 사진 메모 — 기억을 검색에 합류

- **유저 스토리** — 개심사 사진을 라이트박스로 보다가 "엄마가 좋아하던 벚꽃길"이라고 메모를 남긴다. 몇 달 뒤 "엄마 벚꽃"으로 검색하면 이 사진이 올라온다.
- **화면 흐름** — 라이트박스 상세 패널에 메모 영역(textarea + 저장) → 저장 토스트. 메모 있는 사진은 상세에 메모 노출. 검색(S2) 결과에 메모 매칭 사진이 자연 합류.
- **시스템 동작** — `PUT /api/photos/{id}/note {text}` → `notes` upsert(사진별 1메모) + **동기 임베딩**(qwen3-embedding:8b → Chroma `eddr_note_text_v1` 별도 컬렉션, `embeddings.kind='note_text'`) — ollama 다운 시 저장은 성공하고 `embedded:false` 반환(추후 재임베딩). 검색 융합은 `semantic_search_photos`의 note leg(RRF 합류, 임베딩-only — 한국어 메모라 FTS porter 부적합).
- **수용 기준** — 메모 저장 → 메모 단어로 검색 시 해당 사진이 결과 포함(M5 게이트). 골든셋 10/10 무손상 회귀(note leg가 기존 순위를 깨지 않음). ollama 다운 상태 저장 → embedded:false 경로 동작.

### S6. 인덱싱 상태 (v1 S5 계승)

- **유저 스토리** — 새 사진을 잔뜩 찍고 온 뒤, 어디까지 검색 가능한지 확인하고 싶다.
- **화면 흐름** — 배지 "9,218/11,689 검색 가능" → 상태 화면(meta/caption/trip 단계 분해 + 경로 헬스 + **ollama 헬스**). v2 웹도 인덱싱은 **읽기 전용** — 갱신은 `eddr update` CLI 위임(웹의 쓰기는 위치 지정·메모 2종뿐).
- **시스템 동작** — `GET /api/status`(`indexing_stats()` + path_health + ollama reachable).
- **수용 기준** — status 수치가 SQLite 실측과 일치. ollama 다운이 status에서 사전 가시화.

### S7. 모바일 + Tailscale HTTPS (횡단 시나리오)

- **유저 스토리** — 밤에 침대에서, 외출 중 LTE로 폰 브라우저로 접속한다. 현위치도 떠야 한다.
- **화면 흐름** — S1~S5 전부 폰 1열 레이아웃(지도 전폭 + bottom sheet + 드로어). 320px 썸네일 우선 로드.
- **시스템 동작** — `tailscale serve`로 **HTTPS** 종단(secure context 충족 → geolocation 허용) + 외부 접속 동시 해결(v1 M4 흡수). 서버 가드는 ADR-0008 불변(기본 루프백·공개 직노출 금지).
- **수용 기준** — iPhone Safari 실기기(tailnet HTTPS)에서 S1 현위치 → S2 검색 → S4 long-press까지 전 구간 동작. 사진 영역 1화면 전송량 < 1MB.

### S8. (선택) 가족에게 보여주기 (v1 S7 계승)

- **유저 스토리** — 거실에서 가족과 "그때 몽골에서 본 별" — 검색해서 풀스크린으로 함께 본다.
- **화면 흐름** — S2 검색 → 라이트박스 풀스크린 → 자동 넘김 슬라이드쇼.
- **시스템 동작** — 신규 API 없음. 가족은 관람자(계정·권한 없음). 공유 링크는 비포함.
- **수용 기준** — 라이트박스 풀스크린 + 자동 넘김 동작.

### S9. (스케치) 멀티유저 전환 — 맨 마지막 (v1 S8 계승)

수용 기준 없음 — 범위 밖. 전환 시 바뀌는 것: 인증 미들웨어 · 쓰기 경로(위치 지정·메모) 소유자 분리 · 사진 가시성 정책 · 프론트 API 클라이언트 토큰 주입. 전제: [prd.md §6-e](prd.md) "공짜 규율 3개" 유지.

## 5. v1(D25) 대비 계승·폐기

| v1 | 처치 |
|---|---|

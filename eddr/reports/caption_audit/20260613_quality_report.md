# 캡션 절대 품질 평가 보고서 (2026-06-13)

## 0. 요약 판정

- **결론**: "냉면" 검색에 콩나물/숙주 사진이 섞이는 문제는 프롬프트만으로 해결되지 않았고, 더 큰 비전 모델(사진을 읽어 캡션을 만드는 모델)로 재캡션(기존 캡션을 다시 생성)해야 근본적으로 줄어든다.
- **권고**: `p5_grounded`(구체 명사와 매체·프레이밍을 보강한 후보 프롬프트)를 큰 모델(`qwen3-vl:8b` 또는 `gemma4:31b`)에 적용해 음식 등 고위험 카테고리부터 부분 재캡션한다.
- **근거**: 26장 평가셋에서 기존 production(현재 기본 검색 경로) 조합 `gemma4:e2b`/`p3_hybrid`는 평균 7.12점·치명결함 4건이었고, `qwen3-vl:8b`와 `gemma4:31b` 계열은 평균 9.38~9.50점·치명결함 0건이었다.
- **한계**: 이 평가는 통계적 일반화가 아니라 재캡션 의사결정을 위한 방향 신호다. 표본은 26장이고, Opus 단일 채점이며, production 기본값은 이 보고서만으로 변경하지 않았다.

## 2. 배경

골든셋 평가 중 **"냉면" 검색에 "콩나물/숙주" 사진이 섞이는 현상**을 확인했다. 발단은 로컬 비전 모델 `gemma4:e2b`가 production 기본 프롬프트 `p3_hybrid`로 콩나물을 `noodles`로 오캡션한 것. 이를 캡션 품질 결함의 한 증상으로 보고, **여러 방법론(모델 체급·프롬프트)이 만드는 캡션의 절대 품질**을 Opus가 사진과 직접 대조해 평가했다.

## 3. 방법

- **평가셋**: 범용 층화 26장 (food 6·people 5·landscape 5·document 5·store_product 5), 음식에 콩나물·떡 오캡션 회귀 3장 포함. (`20260613_quality_eval_set.json`)
- **채점**: Opus가 사진을 직접 열람해 rubric(일관된 채점표) 5차원(정확성·충실성·환각없음·구체성·형식/privacy, 각 0–2, 합 0–10) + 치명결함(오분류·환각) 플래그를 기록했다. (`20260613_quality_scores.json`)
- **1차 매트릭스 (ablation 4셀)**: ablation은 한 번에 한 요인만 바꿔 원인을 분리하는 비교다. 여기서는 프롬프트 효과와 모델 체급 효과를 분리했다.
  - C0 `gemma4:e2b`/`p3_hybrid` (baseline·현 production)
  - C1 `gemma4:e2b`/`p4_grounded` (프롬프트 효과)
  - C2 `qwen3-vl:8b`/`p3_hybrid` (모델 효과)
  - C3 `gemma4:31b`/`p4_grounded` (체급+프롬프트)
- **2차 개선 루프 (2셀)**: 1차에서 드러난 `p4_grounded` 약점(과소특정·메타맥락 누락)을 보정한 `p5_grounded`를 큰 모델에 적용 — C2b `qwen3-vl:8b`/`p5_grounded`, C3b `gemma4:31b`/`p5_grounded`. (`20260613_quality_p5_compare.json`)
- **프롬프트 이름**: `p3_hybrid`는 현 production 기준 프롬프트, `p4_grounded`는 오분류·환각을 줄이려는 grounding 후보, `p5_grounded`는 p4의 과소특정 문제를 줄이도록 구체 명사와 매체·프레이밍 명시를 보강한 후보로 읽는다.

## 4. 1차 결과

| 유형 | C0 e2b/p3 | C1 e2b/p4 | C2 qwen8b/p3 | C3 gemma31b/p4 |
|---|---|---|---|---|
| food | 5.83 | 6.67 | 9.33 | 9.50 |
| people | 6.60 | 6.20 | 8.80 | 10.00 |
| landscape | 8.40 | 8.40 | 9.40 | 9.60 |
| document | 7.40 | 6.60 | 9.60 | 8.80 |
| store_product | 7.60 | 7.60 | 9.80 | 9.60 |
| **전체 평균** | **7.12** | **7.08** | **9.38** | **9.50** |
| **치명결함 건수** | **4** | **3** | **0** | **0** |

**발견**
1. **모델 체급이 압도적 결정 요인.** qwen8b·gemma31b가 gemma-e2b를 전 유형에서 능가, 치명결함 0 vs 4. 콩나물·떡 noodle 오분류는 큰 모델에서 완전히 사라짐.
2. **프롬프트(p4)는 gemma-e2b에서 무효~역효과** (C0 7.12 → C1 7.08). 작은 모델은 grounding 지시 수행 용량 부족 — 요소 누락(풍력터빈·사람)·새 환각(A/S센터→"식당메뉴")을 유발. → **"프롬프트 변경" 가설 기각, "모델 체급" 가설 입증.**
3. **도메인별 모델 강점 분기**: food/people 메타맥락(화면캡처·거꾸로·성별·인원)은 gemma31b, 텍스트 OCR(document)은 qwen8b 우위(9.60 vs 8.80).
4. **p4 grounding의 약점**: 큰 모델도 명백한 것을 과소특정(salmon→fish, soju→beverage)해 검색 친화성 저하.

## 5. 2차 결과 (p5_grounded 개선 루프)

`p5_grounded` = `p4_grounded` + "명백히 식별 가능하면 구체 명명, 모호할 때만 일반화" + "매체·프레이밍(화면촬영·스크린샷·거꾸로) 명시".

| 셀 | 변경 | 핵심 개선 |
|---|---|---|
| gemma31b p4→p5 | food 9.50→~10.0 | 연어→**salmon**, 고기→**pork/chicken**, 소주 **soju+rotated** 명명 |
| qwen8b p3→p5 | people 8.80→~9.8 | 포토샵 화면 **"computer monitor+Photoshop"**, 거꾸로 **"upside-down"** 포착; A/S센터 전화·시간까지 OCR |

- **표적 약점(과소특정·메타맥락) 해결.** 치명결함 0 유지, 콩나물 회귀 없음.
- 종합 점수는 큰 모델이 이미 천장(9.4+)이라 비슷하나, **검색에 결정적인 구체 명사·메타맥락이 복원**되는 질적 개선.
- 미세 트레이드오프: qwen8b가 사진3 떡→"pork belly", 사진26 브랜드 오독. → 추가 프롬프트 개선의 marginal gain은 작아 루프 종료.

## 6. 결론

- **냉면→콩나물 문제의 근본 해결은 모델 교체.** `gemma4:e2b`는 어떤 프롬프트로도 오분류를 못 고쳤고, `qwen3-vl:8b`·`gemma4:31b`는 정확했다.
- **권고 프롬프트: `p5_grounded`** (큰 모델에서 환각 억제 + 구체성 + 메타맥락의 균형). production 기본은 미변경(평가만).
- **모델 권고 (다음 스텝 입력, 채택은 미결정)**:
  - 개인 사진은 음식·인물·풍경 비중이 커 **`gemma4:31b`+p5**가 품질 최상, 단 느림.
  - 텍스트/스크린샷·문서가 많거나 속도 중요하면 **`qwen3-vl:8b`+p5** (품질 근접 + 2배 빠름, OCR 우위).

## 7. 다음 스텝 — 재캡션 비용 추정 (측정 기반, 단일 호스트)

이 표는 "좋은 모델을 쓰자"를 운영 가능한 결정으로 바꾸기 위한 비용 추정이다. 품질만 보면 큰 모델이 유리하지만, 전량 재캡션은 단일 호스트에서 수일 단위 작업이 될 수 있다.

| 모델 | 실측 throughput | 전량(9,383장) | 음식만(~1,066장 상한근사) |
|---|---|---|---|
| gemma4:e2b | ~7초/장 | ~18시간 | ~2시간 |
| qwen3-vl:8b | ~31초/장 | ~80시간 | ~9시간 |
| gemma4:31b | ~50초/장 | ~130시간 | ~15시간 |

- 전량 재캡션은 큰 모델 기준 수일(단일 호스트). `--remote-host` 듀얼 호스트로 분산 가능.
- **권고 경로**: 검색 오염이 큰 음식 등 고위험 카테고리 우선 부분 재캡션(qwen8b 기준 ~9시간)으로 시작 → 효과 확인 후 전량 결정.
- 재캡션 시 캡션·Chroma(로컬 벡터 저장소) 임베딩 동반 갱신 필요(`_persist_caption` 경로).

## 8. 한계

- 표본 26장은 방향 신호용 — 통계적 결론이 아닌 의사결정 입력.
- Opus 단일 채점(rubric 고정·근거 기록으로 검수 가능).
- privacy: 평가 목적 사진 열람은 사용자 승인 하의 한정 예외(ADR-0001). 산출물엔 photo_id·점수만 보관, 사진 미포함.
- tool/structured output, 검색단 완화(reranker 등)는 이번 범위 밖 — 본 결과로도 품질이 충분하면 불요, 추가 필요 시 다음 단계 후보.

## 9. 용어

| 용어 | 이 보고서에서의 의미 |
|---|---|
| baseline | 비교 기준이 되는 현 production 조합 |
| production | 실제 검색 품질에 영향을 주는 현재 기본 경로 |
| 재캡션 | 기존 캡션을 새 모델·프롬프트로 다시 생성하는 작업 |
| 비전 모델 | 사진을 입력으로 받아 장면·객체·텍스트를 설명하는 모델 |
| grounding | 모델이 사진에서 확인 가능한 근거에 붙어서 쓰도록 하는 지시 |
| OCR | 이미지 안의 글자를 읽어 텍스트로 인식하는 작업 |
| throughput | 단위 시간당 처리량. 여기서는 사진 1장당 처리 시간으로 표현했다. |
| Chroma | 캡션 임베딩을 저장하는 로컬 벡터 저장소 |
